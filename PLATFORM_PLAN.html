1. Executive Summary

What we're building and why.

The Vision A SaaS platform where you onboard any business (law firm, dental clinic, plumber, etc.), deploy a custom AI voice agent for them in minutes, and give them a branded CRM dashboard — or integrate into their existing CRM. You manage everything from a single super admin panel.

Business Model

You are the platform owner. Your clients are businesses that need AI-powered phone intake. Each client gets:

  • A dedicated AI voice agent — with a prompt tailored to their industry, their staff roster, and their brand
  • A branded dashboard — or data pushed to their existing CRM
  • A phone number — calls are answered 24/7 by their AI agent

  • Multi-industry — not locked to legal. Change the prompt template, the dashboard adapts.
  • White-label — each client sees their own brand, not yours.

3. Multi-Agent System

One agent per client, all managed from your backend.

How the Agent Knows Who It Works For

Each client gets a dedicated Retell agent created via the Retell API. The agent's prompt contains that client's specific information — company name, industry, staff roster, services, and intake questions.

Key Concept NOT one agent for all. One agent per client. Your admin panel auto-creates them via Retell's API. All agents send webhooks to your single backend, which routes data by agent_id → firm_id.

Agent Creation Flow

Admin clicks "Create Client" in your panel │ ▼ Your backend calls Retell API: POST https://api.retellai.com/create-agent { "agent_name": "Amy - Bright Smile Dental", "response_engine": { "type": "retell-llm", "llm_id": "..." }, "voice_id": "...", "webhook_url": "https://yourapp.com/api/retell/webhook" } │ ▼ Retell returns: { "agent_id": "agent_abc123" } │ ▼ Your backend saves: firms.retell_agent_id = "agent_abc123" Assigns phone number to this agent via Retell API │ ▼ Agent is LIVE — answering calls with client-specific prompt

Webhook Routing (Single Endpoint, All Clients)

Call to Client A's number → Retell Agent A → webhook → Your server Call to Client B's number → Retell Agent B → webhook → Your server Call to Client C's number → Retell Agent C → webhook → Your server │ agent_id in payload │ ▼ SELECT * FROM firms WHERE retell_agent_id = ? │ ▼ Save lead under firm_id Push to their CRM if configured

Agent Update Flow (Staff Changes, Prompt Edits)

  1. Update staff table in your DB
  2. Re-render the prompt template with current variables (staff names, hours, etc.)
  3. Call Retell API: PATCH /update-agent/{agent_id} with the new prompt
  4. Next call automatically uses the updated prompt

What Happens on "Create Client & Deploy Agent"

  1. Save firm record to firms table in Supabase
  2. Save staff records to staff table
  3. Render prompt template with variables (company name, staff, hours, etc.)
  4. Call Retell API to create a new agent with the rendered prompt
  5. Assign/purchase a phone number via Retell API
  6. Save retell_agent_id and phone number back to firm record
  7. Create login credentials for the client (Supabase Auth)
  8. Send welcome email with dashboard link + login

You POST lead data to a URL the client provides. They wire it to any CRM via Zapier, Make, or n8n.

Level 2: Native Integration (HubSpot, Salesforce — Build Second)

Client clicks "Connect HubSpot" → OAuth popup → authorized → leads flow directly via API. No Zapier needed.

Level 3: Two-Way Sync (Enterprise — Build Later)

Bidirectional: changes in their CRM reflect in your dashboard and vice versa. Only build on demand.

      phone: lead.caller_phone,
      email: lead.caller_email,
      service_type: lead.case_type,
      score: lead.score,
      urgency: lead.urgency,
      summary: call.summary,
      recording_url: call.recording_url
    }
  };

  // Level 1: Generic Webhook
  if (firm.crm_type === 'webhook') {
    await fetch(firm.crm_webhook_url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload)
    });
  }

  // Level 2: Native HubSpot
  if (firm.crm_type === 'hubspot') {
    await fetch('https://api.hubapi.com/crm/v3/objects/contacts', {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${firm.crm_access_token}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        properties: mapFieldsToHubSpot(firm.field_mapping, lead)
      })
    });
  }
}
